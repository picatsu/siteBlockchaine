Projet SorEX

Pourquoi ?

En vous inspirant de la cryptomonnaie Sardex deployée en Sardaigne proposer un système similaire pour les échanges au sein de l'université Sorbonne (le système portera le nom SorEX). Le projet consistera à proposer une architecture client-serveur pour le système SorEX.

Comment ?

Le code sera écrit en JAVA. Version 1.8+ en réseau local avec un système d'exploitation Windows ou Linux

Pour ?

Le projet SorEX crypto entre dans le cadre du projet final associé à l'UE JAVA de la Licence DANT (Développeur d’Applications - Nouvelles Technologies)

Fonctionnalités interessantes

Interface graphique

Nous avons utilisé les bibliothèques graphiques Swing et Awt pour réaliser un front ergonomique et simplifié. Notre logiciel propose plusieurs fonctionnalités, simple d'utilisation à la portée de tous.

Architecture

Mise en place et utilisation d'une architecture Client-Serveur

Facile a utiliser

Notre code a été construit pour éviter au mieux les anomalies éventuelles et optimisé par la même occasion la gestion de la mémoire.

Stockage Intelligent

Le stockage des données se fait par création/lecture/écriture à partir d’un fichier
